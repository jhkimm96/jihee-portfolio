## 왜 저장공간을 따로 관리하나

Pod는 기본적으로 **임시적**이다 — 죽었다 다시 뜨면 컨테이너 안에 있던 파일은 다 날아간다. 근데 DB는 데이터가 사라지면 안 된다. 그래서 "이 Pod가 죽었다 살아나도 계속 남아있는 저장공간"을 따로 관리해야 하고, 여기 두 오브젝트가 쓰인다.

> "PersistentVolume(PV)은 관리자가 미리 준비하거나 StorageClass로 동적으로 준비한, 클러스터 안의 저장공간 조각이다" ... "PersistentVolumeClaim(PVC)은 사용자가 저장공간을 요청하는 것이다"

**비유**: PV는 관리자가 미리 지어둔 **창고**고, PVC는 "저장공간 20GB 주세요"라는 **신청서**다. Pod는 신청서(PVC)만 들고 있으면, 실제로 어느 창고(PV)를 쓰는지는 몰라도 된다.

PromptHub의 PostgreSQL 실제 파일이다:

```yaml
# postgres-pv.yaml — 실제 저장공간
kind: PersistentVolume
spec:
  capacity:
    storage: 20Gi
  persistentVolumeReclaimPolicy: Retain
  local:
    path: /var/lib/prompthub/postgres   # 서버의 실제 폴더
  nodeAffinity:
    required:
      nodeSelectorTerms:
        - matchExpressions:
            - key: prompthub.io/node-pool
              values: [control-stateful]

# postgres-pvc.yaml — 저장공간 신청서
kind: PersistentVolumeClaim
spec:
  storageClassName: local-storage
  selector:
    matchLabels:
      prompthub.io/volume: postgres
  resources:
    requests:
      storage: 20Gi
```

PromptHub는 클라우드 디스크(EBS) 대신 **EC2 서버 자체의 폴더**(`/var/lib/prompthub/postgres`)를 저장공간으로 쓴다. 그래서 `nodeAffinity`로 "이 PV는 반드시 Medium 서버(`control-stateful`)에서만 쓸 수 있다"고 못 박아둔 것이다 — 실제 폴더가 그 서버에만 있으니까다.

`persistentVolumeReclaimPolicy: Retain`은 "PVC가 지워져도 실제 데이터 폴더는 자동으로 안 지운다"는 안전장치다.

## Secret과 ConfigMap — 뭘 감추고 뭘 안 감추나

둘 다 "컨테이너 이미지 안에 값을 직접 박아넣지 않고 밖에서 넣어주는" 방식이다. 차이는 딱 하나, **민감한 값이냐 아니냐**다.

> "ConfigMap은 민감하지 않은 데이터를 key-value로 저장하는 오브젝트다" ... "환경별 설정을 컨테이너 이미지로부터 분리해서, 애플리케이션을 쉽게 이식 가능하게 만든다"

> "Secret은 비밀번호·토큰·키 같은 소량의 민감한 데이터를 담는 오브젝트다" ... "ConfigMap은 비밀을 보장하지 않는다 — 민감한 데이터라면 ConfigMap 대신 Secret을 써라"

**왜 이미지에 직접 안 박아넣나**: 같은 이미지를 개발 환경, 운영 환경에서 그대로 재사용하고 싶은데, DB 주소나 비밀번호는 환경마다 다르다. 이미지 안에 박아두면 환경 바뀔 때마다 이미지를 새로 빌드해야 한다. 밖에서 주입하면 **같은 이미지를 그대로 두고 값만 바꿔** 여러 환경에 쓸 수 있다.

PromptHub의 `product-service` 배포 파일 실제 부분이다:

```yaml
env:
  - name: POSTGRES_PASSWORD
    valueFrom:
      secretKeyRef:
        name: postgres-secret
        key: PRODUCT_SERVICE_PASSWORD
  - name: AWS_REGION
    valueFrom:
      secretKeyRef:
        name: product-secret
        key: AWS_REGION
```

파일 어디에도 실제 비밀번호 값이 안 보인다 — `postgres-secret`이라는 Secret **객체 안에서 이 key를 찾아서 넣어라**는 참조만 있다. 실제 값은 Medium 서버에 별도 파일로만 존재하고, Git에는 절대 안 올라간다.
